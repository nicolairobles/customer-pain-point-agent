# Story 1.4.2 – Implement Query Input Component

## Purpose
Develop the Streamlit component that captures user pain point queries with validation feedback.

## Acceptance Criteria
- [x] Component provides multiline text area with placeholder guidance and word counter.
- [x] Enforces 1–50 word constraint, showing inline validation messages.
- [x] Emits clean string (trimmed, normalized whitespace) for downstream processing.
- [x] Includes optional presets or example queries accessible via UI.
- [x] Unit tests cover validation logic (using Streamlit testing utilities or function extraction).

## Test Criteria
- [ ] **Manual Test**: Enter queries below minimum, above maximum, and within range; verify feedback.
- [x] **Unit Tests**: Run automated tests asserting validator raises appropriate errors.
- [ ] **UX Review**: Confirm component renders correctly in mobile viewport using Streamlit's responsive mode.
