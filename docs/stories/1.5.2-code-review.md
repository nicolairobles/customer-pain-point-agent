## Purpose
Ensure code quality, maintainability, and adherence to project standards through structured peer review.

## Acceptance Criteria
- Pull request template completed with summary, testing evidence, and checklist.
- At least two reviewers approve with comments addressed or resolved.
- Static analysis (lint/format) passes prior to review completion.
- Documentation updates verified alongside code changes.
- Review notes captured in issue tracker for traceability of decisions.

## Test Criteria
- **Process Audit**: Verify PR includes links to relevant stories and tests.
- **Tooling Check**: Confirm lint/format hooks run automatically (pre-commit or CI) during review.
- **Retrospective**: After merge, conduct brief review retro capturing lessons learned.
