How it works (and why it’s reliable)

One orchestrated path keeps validation and provenance intact from intake to report so every claim is traceable and resilient to flaky providers.

1 Intake

Streamlit UI enforces sane query length and preset selection.

2 Orchestrate

LangChain sequences tool calls, parallelizes when possible, and streams progress.

3 Retrieve

Reddit (PRAW) and Google Custom Search gated by feature flags and timeouts.

4 Aggregate

Normalize fields, dedupe near-duplicates, keep URLs, subreddits, timestamps.

5 Extract

OpenAI prompt emits schema-validated JSON; malformed outputs rejected via Pydantic.

6 Report

JSON + narrative surfaced together with citations and runtime metrics.

What you get

Outputs stay concise and decision-ready: a skimmable narrative, aligned JSON, and citations that point back to the exact threads so PMs can defend every call.

The report view makes it easy to defend product decisions: runtime, sources searched, and citations sit next to the pain points.

Engineering decisions

  • Agentic orchestration: LangChain with streaming progress keeps users informed.
  • Contracts: Pydantic around tool payloads and extracted pain points to prevent drift.
  • Normalization + dedupe: preserves citations across merged Reddit/web sources.
  • Feature flags: guard external providers so runs degrade gracefully when creds are missing.
  • Delivery: Streamlit for fast UI iteration; containerized build ships via GitHub Actions to GHCR.
  • Quality gates: tests cover tools, extractors, orchestrator behaviors; healthcheck gates deploys.

Run & deploy

  • Local: Streamlit app runs in a venv; .env.example plus healthcheck cover keys and reachability.
  • Containerized: GitHub Actions builds/pushes to GHCR; Streamlit Community Cloud hosts the public demo.
  • CI: Pytest + healthcheck gate merges and deploys.

Future improvements

  • Source expansion: add review sites and app store feeds with source-aware normalizers.
  • Evaluation harness: offline scoring and regression checks for extraction quality.
  • Active learning: collect reviewer feedback to refine prompts and schemas.
